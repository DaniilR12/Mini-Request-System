# Mini Request System

Невеликий веб-додаток для подачі та обробки заявок. Додаток підтримує дві ролі користувачів — звичайний користувач та менеджер з різними можливостями.

## Можливості

### Для користувача (User)
- Створювати нові заявки з назвою та описом
- Переглядати список своїх заявок та їхні статуси
- Валідація полів форми перед отправкою

### Для менеджера (Manager)
- Переглядати всі заявки в системі
- Змінювати статус заявок (new → in progress → done)
- Фільтрувати заявки за статусом (всі, нові, в процесі, завершені)
- Переглядати повну інформацію про кожну заявку (ID, назва, опис, статус)

## Як це працює

Додаток розділений на дві основні ролі. При запуску можна обрати роль за допомогою кнопок у верхній частині додатка. 

Коли ви обираєте **User**:
- Отримуєте доступ до форми для створення нових заявок
- Бачите список своїх заявок з їхніми статусами
- Форма має базову валідацію — не дозволяє відправляти пусті поля

Коли перемикаєтесь на **Manager**:
- Бачите всі заявки в системі з їхніми ID
- Можете змінювати статус кожної заявки (просто натискаєте кнопку "Change status")
- Статус змінюється по цепочці: new → in progress → done
- Можете фільтрувати заявки за статусом за допомогою випадаючого меню

## Технічна реалізація
**Стек технологій:**
- React 19
- Redux Toolkit (управління глобальним станом)
- Vite (збірка та розробка)
- CSS Modules (стилювання)

**Архітектура:**
- Компоненти розділені за функціональністю в папці `src/components`
- Views (представлення для кожної ролі) у `src/views`
- Redux store в `src/store` з використанням slices
- Кожен компонент має власний CSS модуль для ізоляції стилів

**Збереження даних:**
- Усі дані (заявки, поточна роль, активний фільтр) зберігаються в локальному сховищі браузера (LocalStorage)
- При перезагрузці сторінки всі дані та стан буде збережені
